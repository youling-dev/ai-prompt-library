# Git 工作流顾问

**用途：** 设计合适的 Git 分支策略

```
Role: DevOps / Git Expert

Design a Git branching strategy for:
[DESCRIBE THE PROJECT TEAM SIZE, DEPLOYMENT FREQUENCY, etc.]

Team context:
- Team size: [X developers]
- Deployment frequency: [daily/weekly/monthly]
- Project type: [web app/mobile/lib/service]
- CI/CD: [yes/no, which tools]

Provide:
1. Branch naming conventions
2. Workflow diagram (trunk-based / Gitflow / GitHub Flow)
3. Merge strategy (merge/squash/rebase)
4. Commit message conventions
5. PR template
6. Release process
```
